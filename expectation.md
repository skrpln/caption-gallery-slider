
# Obsidian Gallery Plugin 

## 1. Цель проекта

Создать плагин для Obsidian, который превращает код-блок `gallery` в интерактивную галерею изображений и видео. Плагин должен выглядеть и ощущаться как встроенная функция Obsidian: адаптироваться под активную тему, поддерживать Markdown-подписи со ссылками и hover preview, работать стабильно в режимах Reading и Live Preview. Плагин должен ощущаться **быстрым, лёгким и нативным**, работать стабильно и предсказуемо даже в галереях со сотнями элементов. 

Плагин будет опубликован как open-source в каталоге community-плагинов Obsidian.

## 2. Базовое поведение

- Плагин регистрирует процессор код-блоков для языка `gallery` через `registerMarkdownCodeBlockProcessor`.
- При рендеринге код-блок заменяется горизонтальным виджетом галереи во всю ширину заметки.
- Виджет показывает изображения и видео из одного или двух источников:
  - папка (`dir`);
  - явный список файлов (`list`);
  - оба источника одновременно (объединённый набор).
- Пользователь переключает элементы стрелками влево/вправо.
- У каждого элемента может быть подпись в формате Markdown, которую можно редактировать прямо в интерфейсе.
- Подписи хранятся в отдельных markdown-заметках и создаются по принципу **lazy creation** (см. §5).

## 3. Синтаксис код-блока

Все параметры задаются внутри fenced code block с тегом `gallery`.

```gallery
gallery_id: my_vacation
dir: Attachments/vacation_2026
list:
  - Attachments/special.png
sort: name        # name | created | modified  (по умолчанию: name)
grid: 1,1         # строки, столбцы              (по умолчанию: 1,1)
height: 400       # высота виджета в px          (по умолчанию: ~1/3 viewport)
navigation: plane # plane | preview              (по умолчанию: plane)
```

**Правила параметров**
- `gallery_id` — **обязательный**. Используется как namespace для caption-заметок.
- Должен быть указан хотя бы один из параметров `dir` или `list`. Если ни одного нет — в виджете показывается краткая справка по синтаксису.
- `grid` задаётся как `строки,столбцы`. Максимум: `5,7`.
- При `grid`, отличном от `1,1`, верхняя панель принудительно переключается в режим ползунка (без миниатюр), а пролистывание идёт целыми группами.
- `height` задаёт высоту виджета. Изображения и видео, превышающие эту высоту, обрезаются так, чтобы заполнить область. То же правило применяется к отдельным ячейкам при `grid != 1,1`.

## 4. Спецификация интерфейса

### 4.1. Layout
- Горизонтальный блок во всю ширину заметки с фиксированной высотой (регулируется через `height`).
- Симметричные внутренние отступы между границей виджета и всеми элементами (стрелки, верхняя панель, область подписи).
- Кнопка разворачивания галереи на полный экран в левом верхнем углу.
- Цвета, шрифты и обводки используют CSS-переменные Obsidian — виджет автоматически адаптируется к светлой/тёмной теме.

### 4.2. Верхняя панель — навигация

Два режима, управляемые параметром `navigation` (работают только при `grid: 1,1`):

| Режим                    | Поведение                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **plane** (по умолчанию) | Минималистичные точки или ползунок. Точки = дискретные кружочки, количество равно числу элементов; текущий выделяется большим размером. При более чем 10 элементах точки заменяются ползунком прокрутки (толщина линии = диаметр точки, ползунок = кружок в обводке). Клик по линии или перетаскивание ползунка мгновенно переключает на соответствующий элемент. |
| **preview**              | Полоска квадратных миниатюр изображений/видео. Если миниатюры не помещаются по ширине — появляется собственная горизонтальная прокрутка (колёсико мыши / тачпад). Клик по миниатюре мгновенно переключает на этот элемент. **Доступен только при `grid: 1,1`.**                                                                                                   |

При `grid != 1,1` верхняя панель всегда работает как ползунок, независимо от `navigation`.

### 4.3. Центральная зона — основной viewport
- Показывает текущее изображение или группу изображений (при `grid != 1,1`).
- Стрелки влево/вправо для пролистывания.
- При  `grid != 1,1` пролистывание осуществляется группами.
- Для **видео** внутри viewport накладываются элементы управления в виде минималистичных кнопок, расположенные в нижней части зоны занимаемой видео, перечислены слева направа:
  - play / pause 
  - mute / unmute, 
  - loop toggle,
  - ползунок прогресса внизу.

### 4.4. Нижняя панель — подпись
- Компактная прокручиваемая область ( высотой ~3 строки по умолчанию, пролистыватеся скроллом).
- Размер шрифта совпадает с основным текстом заметки (настройка vault).
- Рендерит тело caption-заметки как Markdown.
- Поддерживает внутренние ссылки Obsidian (`[[...]]`), hover preview и автодополнение `[[` при редактировании.
- Текст подписи редактируется прямо в интерфейсе; во время редактирования виден курсор.

#### Поведение подписей для групп (`grid != 1,1`)
- Область подписи объединяет тела всех заметок, относящихся к текущей группе.
- Заметки разделяются горизонтальной чертой (`---`).
- Порядок: как при чтении книги — слева направо, сверху вниз, построчно.
- Клик по конкретному изображению в группе мгновенно прокручивает текстовую область к соответствующей подписи и кратко подсвечивает этот фрагмент.

## 5. Система подписей (lazy creation)

Подписи не хранятся внутри код-блока. Они живут в обычных markdown-заметках — их можно редактировать, на них можно ссылаться и просматривать через hover preview, как любые другие заметки Obsidian.

**Алгоритм**
1. Плагин парсит код-блок и строит упорядоченный список медиа-файлов.
2. Для каждого файла вычисляется путь к caption-заметке:
   ```
   {gallery_save_dir}/{gallery_id}/{target_type}-{short_hash}.md
   ```
   - `gallery_save_dir` — настройка плагина (по умолчанию: `gallery-captions/`).
   - `target_type` = `img` или `vid`.
   - `short_hash` — короткий детерминированный хеш пути к файлу. Один и тот же файл всегда получает один и тот же хеш, даже если используется в разных галереях.
3. Если caption-заметка существует — в области подписи рендерится её тело (без frontmatter).
4. Если заметки **нет** — в поле подписи показывается плейсхолдер `_insert caption_`.
5. При первом клике в зону редактирования плагин автоматически создаёт markdown-заметку с минимальным frontmatter (`gallery_id`, `target`, `source_path`).
6. С этого момента область подписи рендерит тело заметки.

**Состояние видео в caption-заметках**
Frontmatter заметки для видео может хранить настройки воспроизведения:
- `autoplay: true|false`
- `muted: true|false`
- `loop: true|false`

Эти значения читаются при появлении видео в viewport и переключаются из UI плеера.

## 6. Поддерживаемые форматы

| Тип | Форматы |
|-----|---------|
| Изображения | JPG, JPEG, PNG, GIF, WebP, SVG |
| Видео | MP4, AVI, MOV, WebM |


## 7. Ограничения и краевые случаи

- показывать уместную информативную подсказку в видежете в особых случаях
	- Пустая директория 
	- Несуществующий путь
	- Ошибка прав доступа 
- Одно и то же изображение в разных галереях может иметь **разные** подписи, потому что путь caption-заметки содержит `gallery_id`.
- Виджет должен корректно отображаться внутри hover pop-up (когда заметку с галереей предпросматривают через `[[ссылка]]`).



## 8. Производительность
- **Lazy loading**. Изображения и видео вне текущего viewport (и соседних элементов) не загружаются заранее. Используй `loading="lazy"` или `IntersectionObserver`.
- **Минимальная мутация DOM**. Не перестраивай весь виджет при каждом переключении. Меняй только то, что изменилось: источник изображения, активная точка, текст подписи.
- **Эффективный доступ к файловой системе**. Кэшируй результат сканирования `dir` на время жизни рендера код-блока. Пересканирование — только при изменении заметки или vault.
